# chrome-devtools-mcp 能力矩阵

本文件只定义一个边界：当前 Skill 在调查阶段可以直接做什么、不能直接做什么、以及哪些结论只能推断。

## 能直接做
- 打开、刷新、等待页面，读取快照、截图和控制台日志
- 读取网络请求列表与请求详情
- 在页面上下文执行 JS，例如用 `evaluate_script` 注入最小观测代码
- 通过 `navigate_page(initScript=...)` 在下一次导航前预注入脚本
- 读取页面内对象、序列化结果、调用参数和 `Error().stack`

## 不能直接做
- 直接获取请求 initiator 或 DevTools 内部发起栈元数据
- 直接确认 TLS 指纹、JA3 或更细的握手参数
- 直接确认 HTTP/2 协商细节或底层连接复用状态
- 像专用逆向调试 MCP 那样提供持久化函数 trace、Hook 管理和专门的逆向分析 API

## 只能推断
- 浏览器内成功、页面外失败是否由连接层特征导致
- 某个协议层差异是否真正参与服务端校验
- 某个反调试或环境检测是否同时影响页面行为和服务端验签

## 使用规则
- 能直接观察的，才写成证据。
- 不能直接观察的，不要写成已确认事实。
- 只能推断的，必须标记为风险、差异或待验证结论。
